# D2A2 + MLOps 项目背景

本项目从一个典型的视觉感知需求出发：在低成本深度采集条件下，获得更高分辨率、更清晰、更稳定的深度图结果。围绕这个需求，项目需要解决两层问题：

- 模型层面：如何利用 RGB 图像和深度先验，把低分辨率深度图恢复成高质量深度图。
- 工程层面：如何让训练、验证、测试、调参和模型选择过程可追踪、可复现、可比较。

因此，D2A2 在项目中更适合作为参考的核心模型方案，而 MLOps 则用于把传统机器学习实验流程组织成工程化闭环。

整体逻辑可以概括为：

```text
高质量深度重建需求
    |
引导深度超分辨率任务
    |
参考 D2A2 构建核心模型训练流程
    |
传统训练 / 验证 / 测试 / 调参流程
    |
引入 MLOps 做实验管理和工程化闭环
```

## 1. 需求背景：低成本深度采集与高质量深度图之间的矛盾

深度图能够提供场景中的空间几何信息，在三维重建、机器人感知、自动驾驶、AR/VR、语义分割和场景理解等任务中都有重要价值。很多视觉系统不只需要知道图像中“有什么”，还需要知道物体之间的空间位置、边界轮廓和距离关系。

但在实际采集场景中，高质量深度图并不容易获得。受传感器成本、成像原理、硬件分辨率和环境噪声影响，消费级或低成本深度传感器采集到的原始深度图通常分辨率较低，边界不够清晰，还可能包含噪声和缺失区域。这类低质量深度图很难直接满足高精度视觉任务的需求。

相比之下，高分辨率 RGB 图像更容易获取。同一场景下，RGB 图像和深度图虽然属于不同模态，但在物体轮廓、边缘结构和场景布局上具有一定对应关系。因此，一个自然的技术思路是：利用高分辨率 RGB 图像作为引导信息，帮助低分辨率深度图恢复出更清晰的高分辨率深度结果。

这对应的任务就是引导深度超分辨率（Guided Depth Super-Resolution, GDSR）。它的目标不是简单插值放大深度图，而是结合 RGB guidance、低分辨率 depth 以及其他可能的深度先验，尽量恢复准确的边界、结构和细节。

## 2. 需求拆解：不只是模型效果，还要管理实验过程

从任务本身看，我们首先需要一个能够处理跨模态信息的模型。因为 RGB 和 depth 的数据分布不同，传感器视角和空间位置可能存在偏差，RGB 中还包含大量与深度无关的纹理信息。如果直接把 RGB 特征和 depth 特征融合，可能会带来边界错位、纹理过度转移或深度结构被干扰的问题。

因此，模型侧的需求可以概括为：

- 能利用高分辨率 RGB 图像中的边缘和结构信息。
- 能保留低分辨率深度图中的真实深度观测。
- 能引入额外深度先验，增强物体相对位置和边界信息。
- 能缓解 RGB 与 depth 之间的模态差异、几何错位和纹理干扰。

但只解决模型结构还不够。一个完整的机器学习项目还需要反复进行数据准备、参数配置、训练、验证、测试、调参和模型选择。只要实验数量增加，就会出现另一个层面的需求：如何管理这些实验过程和结果。

工程侧的需求可以概括为：

- 每次实验使用的代码、数据和参数需要可追踪。
- 训练 loss、验证 RMSE、测试 RMSE 需要可查询和可比较。
- best checkpoint 需要和对应的训练过程、测试结果绑定。
- 多组超参数实验需要有统一的 trial 记录和模型选择依据。
- 最终模型产物需要为后续推理或服务化保留清晰接口。

这也是本项目从“训练一个模型”进一步走向“建设一个 MLOps 实验流程”的原因。

## 3. 模型方案：参考 D2A2 完成核心训练任务

在模型方案上，本项目参考 D2A2 来完成 guided depth super-resolution 的核心训练任务。D2A2 的输入形式与上述需求比较匹配：

```text
HR RGB image
LR depth map
monocular depth estimation pseudo-depth
```

其中，RGB 图像提供边缘和结构线索，低分辨率深度图提供真实深度观测，单目深度估计结果提供额外的相对深度先验。D2A2 的“先对齐、再融合”思路，也正好对应了跨模态融合中的几个关键问题：分布不一致、空间几何错位和 RGB 纹理干扰。

所以，D2A2 在这里承担的是“核心模型方案”的角色。它帮助我们回答“用什么模型来完成高质量深度重建”这个问题；而后续 MLOps 流程要回答的是“如何让这套模型实验过程稳定、可复现、可比较”。

## 4. 传统机器学习流程的问题

确定模型方案之后，项目自然进入传统机器学习流程：

```text
准备数据
    |
配置模型和训练参数
    |
训练模型
    |
验证并保存 checkpoint
    |
测试模型效果
    |
多次调参对比
    |
选择最终模型
```

这个流程可以跑通实验，也可以得到 RMSE、checkpoint 和测试结果。但如果只依赖命令行参数、本地日志、手工记录和目录命名，实验规模稍微增加，就会暴露出明显的管理问题。

第一，参数和结果容易分散。学习率、batch size、loss、scale、数据路径、训练日志、验证指标和 checkpoint 可能分别保存在脚本参数、日志文件和结果目录中，后续很难快速确认某个结果来自哪一组配置。

第二，训练、验证和测试容易脱节。训练阶段保存 best checkpoint，测试阶段再手动指定模型路径。如果实验较多，就可能出现 checkpoint 路径混乱、测试结果和训练过程对不上、最终结论难复核的问题。

第三，数据版本难以说明。深度超分辨率任务对数据目录、伪深度文件、划分方式和预处理结果非常敏感。只记录一个 `dataset_dir` 字符串，并不能说明实验实际使用的是哪一个数据快照。

第四，调参过程缺少统一比较。学习率、batch size、loss 类型和学习率衰减策略都会影响最终 RMSE。如果手工运行多组实验，后续很难系统比较每个 trial 的参数、训练曲线、checkpoint 和测试指标。

第五，模型产物不便于后续封装。D2A2 的模型调用不是单输入，而是三输入：

```python
model(guidance, target, mde)
```

如果后续希望把模型作为统一接口调用或服务化，仅保存一个本地 checkpoint 还不足以表达完整推理流程。

因此，本项目的工程重点不是停留在“模型能跑通”，而是把传统机器学习流程升级为一个可管理的实验工程流程。

## 5. 工程目标：引入 MLOps 形成实验闭环

基于上述问题，本项目引入 MLflow、DVC 和 Optuna，把传统训练流程组织成更完整的 MLOps 闭环：

```text
Git 记录代码版本
    |
DVC 记录数据版本
    |
D2A2 执行训练 / 验证 / 测试
    |
MLflow 统一记录参数、指标、日志和模型产物
    |
Optuna 组织超参数搜索
    |
根据 RMSE 选择最优模型
    |
PyFunc 补充模型推理封装
```

各工具在流程中的定位如下：

- Git：记录代码和配置版本，说明实验基于哪一版工程代码。
- DVC：管理数据目录版本，说明实验使用哪一个数据快照。
- MLflow：统一记录参数、指标、checkpoint、日志和模型产物。
- Optuna：组织多组超参数搜索，并把每个 trial 纳入统一比较。
- PyFunc：封装 D2A2 的三输入推理接口，为后续模型服务化提供基础。

这样，项目最终能够回答的不只是“模型 RMSE 是多少”，还包括：

- 这个结果来自哪份代码、哪份数据和哪组参数。
- 训练过程中 loss 和验证 RMSE 如何变化。
- 哪个 checkpoint 被选为 best model。
- best model 在测试集上的表现如何。
- 多组调参实验中，为什么选择当前这一组模型。

## 6. 汇报中的自然过渡讲法

汇报时可以按下面这段逻辑从背景过渡到工程实现：

```text
我们的需求是获得更高质量的深度图。低成本深度传感器采集到的深度图通常分辨率低、噪声多、边界不清晰，而 RGB 图像更容易获得，并且包含丰富的物体边缘和场景结构信息。因此，我们选择 guided depth super-resolution 作为任务方向，希望利用 RGB 图像引导低分辨率深度图恢复出更清晰的高分辨率结果。

为了实现这个任务，我们参考了 D2A2 作为核心模型方案。它同时使用 RGB、低分辨率深度图和单目深度估计先验，并通过先对齐再融合的方式处理跨模态差异、几何错位和纹理干扰问题。

但在真正做实验时，问题不只是选一个模型。传统机器学习流程还包括数据准备、参数配置、训练、验证、测试、调参和模型选择。这个流程手工也能跑通，但当实验变多后，代码版本、数据版本、参数、checkpoint、训练曲线和测试指标会分散在不同地方，最终很难复盘为什么选择某个模型。

所以我们在 D2A2 训练流程外引入 MLOps 思路：用 Git 和 DVC 绑定代码与数据版本，用 MLflow 统一管理实验参数、指标和模型产物，用 Optuna 组织超参数搜索，最后根据统一的 RMSE 指标选择最优模型。这样项目就从单次模型训练，扩展成了一个可追踪、可复现、可比较的实验闭环。
```

这段过渡的重点是：先说明深度重建需求，再说明为什么参考 D2A2，最后自然引出为什么要把传统机器学习流程工程化。

## 7. 一句话定位

可以把本项目概括为：

```text
面向低质量深度图的高质量重建需求，参考 D2A2 构建 guided depth super-resolution 训练流程，并通过 MLflow、DVC 和 Optuna 将传统机器学习实验升级为可追踪、可复现、可比较的 MLOps 闭环。
```
